## 6. 年齡分級（⚠️ 重要：因含社群功能）

本 App 有「社群打卡牆」（使用者可發貼文、留言）＝ **使用者生成內容（UGC）**。
Apple 審查指南 **1.2** 要求含 UGC 的 App 必須具備：

1. 過濾不當內容的機制
2. **檢舉**冒犯內容的功能
3. **封鎖**騷擾使用者的功能
4. 公開的聯絡方式

→ **目前 App 可能缺少「檢舉／封鎖」功能**，這是首次審查最可能被打回的點之一（見交接文件風險清單）。
→ 年齡分級填寫時，UGC 相關題目要如實回答，分級可能落在 **12+**。

## 8. App 隱私「營養標籤」填寫草稿（App Privacy）

App Store Connect → App 隱私，依實際蒐集勾選。本 App 對應如下：

| 資料類型 | 是否蒐集 | 用途 | 是否關聯到使用者身分 |
|----------|----------|------|----------------------|
| 聯絡資訊 - Email | ✅（Google 登入） | App 功能、帳號 | 是 |
| 姓名 | ✅（Google 登入） | App 功能 | 是 |
| 使用者內容（日記/貼文/語音） | ✅ | App 功能 | 是 |
| 使用資料 - 產品互動 | ✅（PostHog） | 分析、改善產品 | 否（匿名分析） |
| 識別碼 - 使用者 ID | ✅ | App 功能 | 是 |
| 診斷資料 | 視 PostHog 設定 | 分析 | 否 |

> 不蒐集：精確位置、聯絡人、健康/健身的 HealthKit 資料、廣告識別碼。
> 語音輸入：錄音傳到伺服器轉文字後不長期保存，標註於「使用者內容」。

# MindGym UI 框架演進與設計藍圖報告

本報告整理了 MindGym 社群與練習框架的設計演進脈絡，並提供最終基於現有專案 **3 Tab** 的介面規劃藍圖。

---

## 第一部分：設計演進脈絡 (Context & Evolution)

我們的討論歷經了數個核心痛點的思辨，最終收斂出目前兼具**心理安全感（PERMA）**、**習慣養成**、**熟人社交**與**商業變現**的平衡方案。

### 1. 初始痛點：如何避免交友軟體化，並整合「頻道」與「小隊」？
* **初始想法：** 原本想如同 Discord 一般，將練習分為「感恩」、「正念」等不同頻道。然而引入 4-6 人「小隊（Squad）」概念後，兩者的關係與能見度產生了衝突。
* **解決方案的演進：** 我們提出了「主題營隊（Camp）模式」（方案 C）。將頻道轉化為限時的營隊（例如：14 天感恩營），用戶報名後自動與另外 5 人組成小隊。這將原本發散的頻道練習，變成了有明確目標的旅程，並作為實體工作坊的精準變現漏斗。

### 2. 進階痛點：短期營隊結束後，如何維持長期習慣？
* **痛點：** 7-21 天的營隊雖然有破關感，但心理健康練習需要終身持續。若營隊結束小隊就解散，用戶的練習可能隨之中斷。
* **解決方案的演進：** 引入 **「雙軌制 (Dual-Track System)」**：
  * **個人花園 (Solo Track - 長期)：** 用戶隨時可記錄，屬於終身私密回憶。
  * **營隊特訓 (Social Track - 短期)：** 營隊期間將個人練習同步至 6 人小隊牆，作為社交放大器。
  * **小隊續約機制：** 營隊結業後，若成員合得來，可選擇續約升級為「常駐遠征隊」，解鎖「跨主題分享」特權（可在小隊內同步正念、情緒等所有練習）。

### 3. 架構痛點：與現有程式碼架構的整合
* **痛點：** 概念設計一度延伸至 5 個 Tab（獨立出個人花園、實體活動等），但這與 MindGym 目前已有的三個路由 (`/app/home`, `/app/community`, `/app/profile`) 偏離過大，會造成開發成本暴增。
* **解決方案的演進：** 我們將雙軌制完美映射回現有的 3 Tab。
  * `home` (訓練中心) = 個人花園（發文輸入點，含同步開關）
  * `community` (社群) = 小隊牆、探索新營隊、實體活動版位（互動與變現）
  * `profile` (個人頁面) = 長期歷史紀錄與 PERMA 數據統計

### 4. 關係深化痛點：如何加強與現實親友、伴侶的連結？
* **痛點：** 隨機配對雖能建立新的支持圈，但無法深化用戶本就擁有的既有關係（如家人、伴侶、現實好友）。
* **解決方案的演進：** 在報名營隊時設計「雙模式入口」：
  * **隨機配對 (Public Squad)：** 與 5 位陌生人練習。
  * **邀請親友 (Private Squad)：** 產生邀請連結，拉現實中的伴侶/親友組成私人小隊（同樣限 6 人），將練習融入真實親密關係中，同時形成病毒式成長 (Viral Loop)。

## 第二部分：最終 UI 介面設計藍圖 (基於 3 Tab 架構)

### 1. 🏋️ Tab 1：訓練中心 (Home - `/app/home`)
* **定位：** 個人花園 (Solo Track)。用戶每天獨立進行「心理練習」的唯一輸入點。
* **UI 佈局：**
  * **今日任務：** 引導用戶進行感恩日記、正念呼吸或情緒記錄。
  * **同步開關 (Sync Toggle)：** 在寫完日記按下「送出」前，若用戶處於活躍小隊中，會顯示：`[v] 同步分享至：星空漫步小隊 (14天感恩營)`。勾選後才會同步至小隊牆。

### 2. 🌍 Tab 2：社群 (Community - `/app/community`)
* **定位：** 社交與變現 (Social Track)。用戶與小隊互動、探索營隊與實體沙龍的儀表板。
* **UI 佈局：**
  * **我的遠征隊 (My Active Squads)：** 置頂卡片。點擊進入 6 人小隊的專屬動態牆（無發文框，僅有共鳴標籤與匿名投遞箱）。
  * **探索營隊與親友同行 (Discover & Invite)：** 中間選單。可報名新營隊，並選擇「隨機配對」或「建立私人連結邀請親友」。
  * **實體沙龍 (Offline Events)：** 底部版位。展示專家帶領的實體工作坊。當小隊結業時，提供「整隊報名」的 Call-to-Action。

### 3. 👤 Tab 3：個人頁面 (Profile - `/app/profile`)
* **定位：** 數據與回顧 (Storage & Stats)。
* **UI 佈局：**
  * **練習歷程：** 整合用戶在訓練中心寫過的所有歷史紀錄，即使營隊解散也永遠留存。
  * **PERMA 雷達圖與成就：** 視覺化呈現心理健康狀態，並展示獲得的營隊徽章與來自隊友的共鳴次數。

# 不會寫程式，也能改網站

給新加入的協作者：從「打開電腦」到「東西真的改到網站上」的完整步驟。

整份手冊只要求你會做一件事：**用中文跟 Claude Code 說話**。每一步旁邊都寫好「跟它說什麼」，照著打、看到它問你要不要允許就按同意就好，不需要懂任何程式碼或指令。

> **開始前確認**：Dennis 需要先把你的 GitHub 帳號加進專案的 collaborator，你才有權限把修改推上去 —— 沒有的話跟他要一下邀請。

---

## 1. 把專案載到你的電腦（只需要做一次）

**a.** 打開「Terminal」（Mac：按 `⌘ + 空白鍵`，輸入 Terminal 按 Enter）。

**b.** 決定要把專案放在哪裡，例如桌面，然後輸入：

```
cd Desktop
claude
```

這會在桌面資料夾裡打開 Claude Code。

**c.** 跟 Claude Code 說：

> 請幫我把 https://github.com/PSY-by-PSY/MindGym.git clone 到目前的資料夾

它會問要不要執行這個指令，按**允許 / Yes**。第一次可能還會請你登入 GitHub 帳號（會跳出一個網頁連結，點進去登入即可）。

**d.** 下載完成後，輸入 `exit` 離開，再進到專案資料夾裡重新打開一次：

```
exit
cd MindGym
claude
```

之後每次要工作，都是先 `cd MindGym` 再 `claude`。

---

## 2. 跟 Claude Code 說出你要改什麼（每次工作的主要步驟）

**a.** 直接用平常講話的方式描述你想改的東西，不用懂程式。例如：

> 首頁「暖身卡牌」那個模組的按鈕文字改成「開始練習」

**b.** 它會先去讀相關檔案，改完後會告訴你改了哪裡。如果它要執行指令或寫入檔案，畫面上會跳出確認，按**允許**就好。

**c.** 想眼見為憑，可以請它直接跑起來給你看：

> 幫我把網站跑起來，我要看一下改完的樣子

它會開一個本機預覽，你可以直接跟它說「這裡怪怪的」「顏色不對」，它會繼續調整。

---

## 3. 改完之後：交出去（確認滿意之後再做）

**a.** 滿意這次的修改後，跟它說：

> 請幫我把這些修改建立一個新分支、commit、推上 GitHub，並開一個 Pull Request

它會自己取分支名稱、寫 commit 訊息、推上去，最後給你一個 Pull Request 連結。

**b.** 把那個連結傳給 Dennis（LINE / Slack 都可以），跟他說「這個改好了，麻煩看一下」。

> **不會弄壞正式網站**：你的修改在 Dennis 按下「merge」之前，都不會影響大家現在看到的正式網站，所以放心大膽地改、試錯完全沒問題。

---

## 給 Dennis：收到 Pull Request 之後的驗收清單

每一個 PR 連結進來，照這個順序過一次：

- [ ] 打開 PR 頁面，看 Vercel 自動生成的 preview 連結，實際點過一次改動的功能
- [ ] 檢查 PR 說明有沒有提到動到 Supabase schema —— 有的話先手動去 Supabase 跑對應的 `.sql`
- [ ] 沒問題就在 GitHub 上按 **Merge**（分支會自動刪除，不用手動清）
- [ ] Merge 後 Vercel 會自動把 main 部署到正式網站，等 1–2 分鐘後回正式網址複查一次

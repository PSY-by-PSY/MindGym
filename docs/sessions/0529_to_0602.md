## Step 3 — InMind 分數小數點精度修正

**檔案：** `src/routes/onboarding.tsx`，`app.py`，`src/components/pretest/ResultsScreen.tsx`

### 根本原因（已找到）
`app.py` line 913–917 在存入 DB 時對分數做了 `round()`，把小數點吃掉：
```python
"p_score": round(scores_dict["P"]),  # 3.7 → 4
```

### 修正方式
1. **app.py**：移除 `round()`，直接存原始 float：
   ```python
   "p_score": scores_dict["P"],
   "e_score": scores_dict["E"],
   # ... 等
   ```
2. **ResultsScreen.tsx**：確認所有分數顯示都用 `.toFixed(1)`（已在 lines 93, 219, 574 有，確認全面覆蓋）
3. **Supabase perma_scores 表**：確認 p_score ~ a_score 欄位的型別是 `float4` 或 `numeric`，而非 `int`（若是 int 需在 Supabase Dashboard 改型別）

**驗證：** API 回傳 3.7，DB 存 3.7，APP 顯示 3.7。

## Step 4 — 測驗流程優化：立即存檔 + 按鈕樣式調整

**檔案：** `src/routes/onboarding.tsx`，`src/components/pretest/ResultsScreen.tsx`

### 4A. API 生成速度（選做／後續優化）
目前使用 `claude-sonnet-4-5`，max_tokens 2500。**推薦的加速方式是改為 Streaming 串流**（前端邊收邊顯示），使用者感知速度大幅提升，不需等全部跑完。此項改動較大，可作為獨立 Step 處理，本次暫不包含。

### 4B. 立即儲存打卡紀錄
目前問題：使用者沒按「保存」就離開，建議就不會留存。

**修正方式：**
- AI 建議跑出來（`screen === 'report'`）的那一刻，**立即寫入感恩打卡紀錄到 DB**（`gratitude_entries` 或新建一個 `inmind_sessions` 表）
- 不需等使用者再點按鈕確認
- 後續使用者在 profile 頁或首頁可以看到這次紀錄

### 4C. 按鈕樣式翻轉（引導使用者按正確按鈕）
**現狀：** ResultsScreen 底部有「分享我的報告」和「開始第一次練習」兩個按鈕。

**修正方式（對應 ResultsScreen.tsx lines 685–841）：**
- **「分享我的報告」** → 改為白色背景、深色文字（次要按鈕樣式）
- **「開始第一次練習 →」** → 改為藍色背景、白色文字（主要按鈕樣式，醒目）
- 邏輯不變，只是視覺樣式對調

**驗證：** 結果頁「開始練習」是藍色主按鈕；完成測驗後 DB 有立即儲存的紀錄，不需按分享。

## Step 14 — Supabase 測試資料清理（手動操作步驟）

**⚠️ 這個步驟需要在 Supabase Dashboard 手動操作，Claude 無法直接處理。**

### 操作步驟
**路徑：** Supabase Dashboard → 你的專案 → SQL Editor

**1. 先 SELECT 確認要刪的資料：**
```sql
SELECT id, user_id, item_1, item_2, item_3, created_at
FROM gratitude_entries
WHERE item_1 ILIKE '%111%' OR item_2 ILIKE '%111%' OR item_3 ILIKE '%111%'
ORDER BY created_at DESC;
```

**2. 刪除對應的 comments 和 likes（先刪 FK 依賴）：**
```sql
DELETE FROM comments
WHERE entry_id IN (
  SELECT id FROM gratitude_entries
  WHERE item_1 ILIKE '%111%' OR item_2 ILIKE '%111%' OR item_3 ILIKE '%111%'
);

DELETE FROM likes
WHERE entry_id IN (
  SELECT id FROM gratitude_entries
  WHERE item_1 ILIKE '%111%' OR item_2 ILIKE '%111%' OR item_3 ILIKE '%111%'
);

DELETE FROM gratitude_item_tags
WHERE entry_id IN (
  SELECT id FROM gratitude_entries
  WHERE item_1 ILIKE '%111%' OR item_2 ILIKE '%111%' OR item_3 ILIKE '%111%'
);
```

**3. 最後刪除 entries 本體：**
```sql
DELETE FROM gratitude_entries
WHERE item_1 ILIKE '%111%' OR item_2 ILIKE '%111%' OR item_3 ILIKE '%111%';
```

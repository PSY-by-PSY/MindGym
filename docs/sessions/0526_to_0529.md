## Step A1 — 感恩日記 INTRO 版面重構

**檔案**：`src/routes/app.gratitude.tsx` → `IntroStage` 組件

**修改內容**：
1. 刪除現有描述文字（「停下來，把注意力放回身邊的美好…」那整段 3-C 區塊）
2. 換成兩個獨立區塊：

   **區塊一（常駐顯示）**：
   > 感恩日記（Gratitude Journal）是正向心理學中最具代表性的練習之一，透過每天有意識地回顧值得感謝的事件，幫助大腦重新聚焦於生活中的支持、善意與美好經驗。

   **區塊二（「查看更多」展開，預設收合）**，展開後依序顯示：
   - **核心目標**
     - 建立覺察生活中的美好以及練習表達感恩的習慣
     - 透過簡單、低負擔的書寫，引導我們開始留意：今天有哪些事情值得被感謝？哪些人、環境與體驗支持了自己？自己是否也值得被感謝？
   - **練前準備**
     - 練習時常：建議每日 5–10 分鐘
     - 時段推薦：建議在 19:00–24:00 之間練習
     - 環境營造：開啟 APP 安神背景音樂、暫停所有訊息通知、找安靜空間
   - **不建議練習的時刻**
     - 情緒極端崩潰時、極度疲憊時
   - **研究支持**
     - 持續感恩練習有助提升：正向情緒、人際連結、意義感、心理韌性、壓力調節與睡眠品質
     - 附上兩篇文獻引用（Choi et al. 2025、Folk & Dunn 2023）

3. 難度 default 從 `'advanced'` 改成 `'basic'`（`GratitudePage` 的 `useState<Difficulty>('advanced')` → `useState<Difficulty>('basic')`）

## Step A5 — 資料存取時機調整

**檔案**：`src/routes/app.gratitude.tsx` → `GratitudePage` + `handleFinalSave` + `SummaryStage`

**現狀**：按下 CelebrateStage 的按鈕 → `handleFinalSave` → 存 DB → navigate

**修改目標**：
- 在 SUMMARY stage 按下「下一步：完成這次練習」時，就立即存 DB
- 進入 CELEBRATE 時資料已存完，`savedEntryId` 已有值
- CelebrateStage 的「結束今天練習」只需執行 navigate，不再存 DB

**實作方式**：
1. 把 `handleFinalSave` 的存 DB 邏輯提取成獨立的 `saveEntry()` async function
2. `SummaryStage` 的「下一步」`onContinue` 回調在 `GratitudePage` 層改成：先呼叫 `saveEntry()`，再 `setStage('CELEBRATE')`
3. CelebrateStage 的 `onNavigate` 簡化成只做 `router.invalidate()` + `navigate`

## Step A6 — 補打卡功能（0→1）

**檔案**：`src/routes/app.gratitude.tsx` → `WritingStage` 組件

**修改內容**：
1. **UI**：在 WritingStage 日期行的旁邊加一個「修改日期」小按鈕
2. **互動**：點擊後彈出底部 sheet，列出兩個選項：
   - 今天（YYYY/MM/DD，星期Ｘ）
   - 昨天（YYYY/MM/DD，星期Ｘ）
3. **狀態**：`WritingStage` 加 `selectedDate: Date` state，預設 `todayDate()`；選昨天就改成昨天
4. **存入**：`isoDate(selectedDate)` 作為 `entry_date`，而非硬寫 `todayDate()`
5. **Streak 即時更新**：補打卡成功後（在 Step A5 的 `saveEntry()` 完成後）重新呼叫 `loadStreak()` 並更新 CELEBRATE stage 的 streak 顯示

> **注意**：需把 `selectedDate` 從 `WritingStage` 往上提升到 `GratitudePage`，才能在 `saveEntry()` 中取到正確日期。

## Step E1 — 社群觸發：完成練習後 + 每日首次進入社群

**檔案**：`src/routes/app.community.tsx` + `src/routes/app.gratitude.tsx`

**觸發條件**：
1. 每次完成感恩練習（從 CELEBRATE navigate 到 `/app/community`）時
2. 每天第一次點擊底部導航「社群」tab 時

**行為**：系統跳出 modal，顯示一則其他使用者的感恩日記（隨機從 `gratitude_entries` 抓一筆 `is_shared = true` 且 `user_id != 自己` 的資料）

**實作方向**：
- 在 community 頁加 `showWelcomeEntry` state
- 條件一：從 gratitude navigate 來時，透過 URL search param（如 `?showEntry=1`）觸發
- 條件二：使用 `localStorage` 紀錄今天是否已顯示過（key: `community_last_shown: YYYY-MM-DD`），每次進入 community tab 時檢查

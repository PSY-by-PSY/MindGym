# 一週回顧(Weekly Review)計劃書 — 簡化定案版

> 入口:「我的健心日記」卡片**底部的小框框** → 點進去開**獨立的週回顧頁**
> 核心目標:**使用者 30 秒內看懂自己這一週的健心狀況**
> 日期:2026-07-12(v3,依討論簡化定案)

---

## 1. 定案範圍(只保留這四樣)

| # | 功能 | 內容 |
|---|---|---|
| 1 | **感恩** | 本週記錄感恩了幾次 |
| 2 | **過程** | 本週過程目標(專注時刻/錦囊/早晨)做了幾次 |
| 3 | **社群留言** | 這週收到的留言原文(**不放按讚數**);預設精選 1–2 則,「展開全部回饋 ▾」看全部 |
| 4 | **AI 週分析** | 感恩的**對象**、常提到的**詞彙(內容主題)**、常出現的**情緒**,加 2–3 句 AI 摘要 |

明確不做(本期):逐日內容原文時間軸、按讚數、連續打卡點條(檔案頁頂端已有 streak)。

## 2. UI 設計

### 2.1 入口小框(「我的健心日記」卡片底部)

```
┌──────────────────────────────────┐
│  (…日曆格子…)                     │
│                                  │
│ ┌──────────────────────────────┐ │
│ │ 🗓 本週回顧                    │ │
│ │   感恩 4 次 · 過程 3 次      › │ │
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```

- 一行摘要(兩個次數直接可見)+ 右箭頭,樣式沿用日曆內「練習紀錄」列(bg-muted 圓角列)。
- 點擊 → 導向 `/app/weekly-review`。
- 本週 AI 分析首次生成完成時,框上顯示小紅點(沿用回顧集的未讀紅點樣式)。

### 2.2 週回顧頁(`/app/weekly-review`)

```
┌──────────────────────────────────┐
│ ‹ 返回        本週回顧            │
│        ‹  7/7 – 7/13  ›          │ ← 可切換上一週(看歷史)
│                                  │
│ ┌────────────┐  ┌────────────┐   │
│ │ 🙏 感恩日記 │  │ 🎯 過程目標 │   │
│ │    4 次    │  │    3 次    │   │ ← 大數字 tile
│ └────────────┘  └────────────┘   │
│                                  │
│ 💬 這週收到的留言                  │
│ ┌──────────────────────────────┐ │
│ │「看到你寫的第三件事,好溫暖…」    │ │
│ │  — 7/10                      │ │ ← 精選留言(1–2 則)
│ └──────────────────────────────┘ │
│ [展開全部回饋 ▾]                  │ ← 展開全部留言原文+日期
│                                  │
│ ✨ AI 週分析                      │
│  感恩對象:▓▓▓▓▓▓░░░ 身邊他人 60%  │ ← 色條沿用對象配色
│  常提到的詞彙:[家人][晚餐][報告]   │ ← chips
│  常見情緒:[平靜 ×3][有成就感 ×2]  │ ← chips
│  💡「這週你的感恩多半來自與家人的   │
│     互動,也常寫到工作上的小成就…」 │ ← AI 2–3 句摘要
└──────────────────────────────────┘
```

- 週切換:預設本週(週一～今日),`‹ ›` 可翻看過去週;未來週不可切。
- 30 秒節奏:兩個大數字(~5 秒)→ 精選留言(~8 秒)→ AI 分析(~15 秒)。
- 設計語彙沿用全站:rounded-3xl bg-card shadow-soft、tile 色塊、`TARGET_COLORS` 對象配色。

### 2.3 空狀態

| 情況 | 顯示 |
|---|---|
| 本週 0 筆練習 | 入口框改顯示「這週還沒有紀錄,寫下第一篇 →」(連到感恩日記);頁面顯示鼓勵空狀態 |
| 0 則留言 | 留言區整段隱藏 |
| 本週 < 2 筆 | AI 區顯示「再多寫幾篇,AI 週分析就會出現」;次數照常顯示 |
| AI 生成中/失敗 | 次數與留言照常顯示,AI 區顯示「分析生成中…」— **頁面永不因 AI 失敗而空白** |

## 3. 資料與後端

### 3.1 前端直查(RLS 已可讀,零後端改動)

| 資料 | 來源 |
|---|---|
| 感恩次數 | `gratitude_entries`(practice_type='gratitude'、本週範圍)count |
| 過程次數 | `focus_logs` + `morning_logs`(本週範圍)count |
| 社群留言 | `comments` where `entry_id ∈ 本週自己的分享貼文`(含留言原文、日期) |
| 感恩對象統計 | `gratitude_entries.target_1..3` 眾數(**免 AI**,沿用 `TARGET_COLORS`/`TARGET_INSIGHT`) |

### 3.2 AI 分析:`POST /api/reviews/weekly-digest`

- 參數:`period_start`(該週週一)。該週紀錄 ≥ 2 筆才生成,否則 409。
- 快取:存 `pro_reviews`(`review_type='weekly_digest'`、`module_id=NULL`);已有該週 digest 且 `entry_count` 未變 → 直接回舊的;變了 → 重生並 UPDATE 同一列(靠既有 UNIQUE 定位)。
- 觸發:進週回顧頁時呼叫(頁面自己 lazy 載入,不掛 `checkAndGenerateReviews()`,流程更單純)。
- 模型:`claude-haiku-4-5`,`meter_claude("weekly-digest", …)` 計量;語氣沿用 BOUBA(溫暖、不批判、不診斷)。
- 輸出 JSON(`content` jsonb,v:1):

```json
{
  "summary":  "2–3 句本週觀察(80 字內)",
  "keywords": ["家人", "晚餐", "報告"],
  "emotions": [{"label": "平靜", "count": 3}, {"label": "有成就感", "count": 2}]
}
```

- 歷史週:`pro_reviews` 已有的直接讀;沒有的過去週若該週有紀錄,同端點即時補生成。
- SQL 遷移:冪等重建 `pro_reviews` 的 CHECK 約束,加入 `'weekly_digest'`。
- 成本:每人每週最多個位數 haiku 呼叫(entry_count 沒變一律回快取),單次 <1k tokens,可忽略。

## 5. 驗收標準

1. 入口框一眼看到兩個次數;點進頁面 30 秒內能回答「這週感恩幾次、過程幾次、有誰留言、AI 說我常感恩誰」。
2. 本週 0/1/2/7 筆等各種資料量下,入口框與頁面皆不破版、不空白。
3. AI 端點失敗或逾時,次數與留言照常顯示。
4. 週切換到任何過去週都正確;未來週不可切;跨週(週一凌晨)正確歸零。
5. 語言切換後所有字串正確翻譯。

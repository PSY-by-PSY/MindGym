## Step 5｜AI 功能升級（感恩對象標記 + 回饋強化）

**對應檔案**：`src/routes/app.gratitude.tsx`（`SummaryStage`）、後端 API
**參考來源**：UX 設計師文件（第三步 AI 標記、第四步 AI 回饋）

### 目前狀況

- AI 回饋：只有摘要（`fetchSummary` → `coach's note`）
- 標籤：社群頁有關鍵字標籤（感受/事件/對象/其他），但非「感恩對象」分類

### 需要修改

#### 5-A｜新增感恩對象標記 API

後端新增 `/api/tag-gratitude-targets` endpoint（或在現有 summary endpoint 一併回傳）。

**標記分類**：
| 代碼 | 類別 | 說明 |
|------|------|------|
| `others` | 👥 身邊他人 | 家人、朋友、伴侶、同事、陌生人；僅有姓名則統一標「身邊他人」 |
| `self` | 🙋 自己 | 努力、堅持、情緒覺察、自我照顧 |
| `environment` | 🌳 環境 | 天氣、空間、大自然 |
| `experience` | ✨ 體驗 | 電影、音樂、美食、旅行 |
| `custom` | 🏷️ 自訂 | 其他 |

**回傳格式**：
```json
{
  "tags": [
    { "item": 1, "target": "others", "label": "同事" },
    { "item": 2, "target": "self",   "label": "自己" },
    { "item": 3, "target": "experience", "label": "體驗" }
  ]
}
```

**AI Prompt 參考**（見 UX 文件）：
```
Role: 心理學分析助手
Task: 根據使用者提供的感恩事件，精準標記感恩對象。
Rule:
1. 識別對象類別：[身邊他人]、[自己]、[環境]、[體驗]、[自訂]。
2. 若提及特定人名或明確關係，標記為該具體類別；若僅有名字，統一標記為「身邊他人」。
```

#### 5-B｜SummaryStage 顯示感恩對象標籤

在每則感恩項目下方加上對應的標籤 badge（如 `👥 同事`、`🙋 自己`）。

#### 5-C｜AI 回饋格式升級

現有 `fetchSummary` 的回饋內容升級為兩段結構：

1. **情緒摘要**：一句話反映使用者情緒狀態，溫柔不批判，強化正向情緒
2. **行動建議**：
   - 若感恩對象含「身邊他人」→ 鼓勵直接分享給對方
   - 一般情況 → 提供生活中可留意的建設性建議

#### 5-D｜DB 儲存感恩對象標記

在 `gratitude_entries` table 新增欄位（或新建 `gratitude_item_tags` table），儲存每則的 `target` 類別，供後續「感恩對象地圖」使用。

#### 6-A｜完成統計資訊
在 PERMA 加分上方加入：
- **今日完成**：已完成 N 次練習
- **連續紀錄**：已連續 N 天 🔥

#### 6-B｜感恩對象地圖（累積圓餅圖）

新的卡片區塊，顯示**累積至今**的感恩對象比例：

- **視覺**：環形圖（donut chart）或圓餅圖
- **資料**：從 `gratitude_item_tags` 累計各 target 的次數與百分比（Step 5D 存入的資料）
- **文字洞察**（一句話，根據佔比最高的類別動態生成）：
  - 身邊他人最高 → 「你的幸福感有很大一部分來自身邊的人...」
  - 自己最高 → 「你非常懂得欣賞自己的努力與成長...」
  - 環境最高 → 「你對生活中的細微美好特別敏感...」
  - 體驗最高 → 「你善於從日常的小體驗中找到喜悅...」
- **「？」按鈕**：點開小彈窗，說明各對象類別的心理學意義與好處（詳見 UX 文件完整說明）

#### 6-C｜分享設定升級

- 預設改為**實名分享**（顯示真實姓名）
- 匿名時使用「能量代號」（如：溫暖的星火、清晨的微風），而非「匿名使用者」
- 分享說明文字更新為：「以實名在社群分享你今天的感恩」

#### 6-D｜完成後三個導航選項

「收下，繼續加油」按鈕後，替換為三個選項：

| 按鈕 | 行為 |
|------|------|
| 💬 點擊留言 | 跳至社群，直接捲動到自己剛分享的那則，並自動展開留言框 |
| 📖 查看更多 | 跳至社群健心打卡牆（無文字雲，直接看牆） |
| ✕ 關閉 | 跳至社群首屏（文字雲 → 往下滑才看到打卡牆） |
